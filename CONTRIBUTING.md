# Contributing Guide

感谢你关注这个项目。

本项目欢迎非商业开源贡献，适合以下类型的改进：

- 透明化工作流与多智能体可视化
- PDF / DOCX 解析稳定性
- 中文化与国际化
- 可访问性、可读性与交互体验
- 章节级分析、冲突图谱与报告导出

## 国际化 (i18n) 贡献指南

本项目支持中英文双语，欢迎贡献更多语言或改进现有翻译。

### 前端 i18n

翻译文件位置：`frontend/src/lib/i18n.ts`

添加新语言步骤：
1. 在 `messages` 对象中添加新的语言代码（如 `ja`、`ko`）
2. 参考 `zh` 或 `en` 的键值对，添加对应翻译
3. 更新 `Locale` 类型定义
4. 在 `setLocale` 函数中添加语言选项

### 后端 i18n

翻译文件位置：`backend/src/lib/activity-i18n.ts`

后端 i18n 主要用于：
- 活动事件标题和消息
- 阶段状态描述
- 智能体状态文本

## 开发流程

1. Fork 仓库并创建功能分支
2. 保持提交粒度清晰
3. 在提交前运行前端构建与后端类型检查
4. 提交 PR，并说明改动目的、验证方式与影响范围

## 本地检查

前端：

```bash
cd frontend
npm run build
```

后端：

```bash
cd backend
npx tsc --noEmit
```

## 提交建议

- 使用清晰的 commit message
- 尽量围绕单一主题提交
- 不要提交真实密钥、数据库文件和本地日志

## PR 建议内容

- 改了什么
- 为什么改
- 如何验证
- 是否影响前端、后端、数据库或环境变量

如果改动涉及 Prisma schema、数据库字段或解析流程，请在 PR 描述中额外说明：

- 是否需要执行 `npx prisma db push` 或其他数据库同步步骤
- 是否会影响历史项目数据的兼容性
- 是否更新了 `README.md` / `CHANGELOG.md` 等说明文档

## 许可说明

本项目仅接受与当前“非商业开源”许可兼容的贡献。提交代码即表示你同意该贡献在项目现有许可下发布。
